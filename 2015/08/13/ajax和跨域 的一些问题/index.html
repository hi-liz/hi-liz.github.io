ajax和跨域 的一些问题

发表于 2015-08-13

ReadyState 的5个值表示的含义：
readyState=0表示xhr对象创建完成
readyState=1表示已经发送了请求
2 浏览器已经收到了服务器响应的数据
3 正在解析数据
4 数据已经解析完成，可以使用了

  • ajax的xhr对象的创建
  • get方法的参数是要拼接到url上的 post的是通过xhr.send()发送的，post要设置响应头xhr.setHeaders();
  • xhr.open(‘get’,url,true)
  • xhr.readyStateonchange

  • xhr.status 的状态
    200表示响应成功
    301 永久重定向
    302临时性重定向
    404没有找到请求的资源
    500服务器端错误

    jsonp的跨域问题 ，因为同源策略的原因，不同域名，协议，端口之间的传输数据造成跨域问题，服务端返回的是函数的回调callback(data)，jsonp的请求方式一定是get

    //封装myJsonp，用来跨域请求
    function crossDomain (url,arg,fn){
    //拼接url
    var queryString =””;
    //arg是对象的key –value for in 循环
    for(var key in arg){

    queryString += key +"="+arg[key]+'&';

    }
    url +=’?’+queryString
    //回调函数的name
    var callbackName =’jsonp_’+Math.random().toString().substr(2);
    url +=’callback=’+callbackName;
    //定义一个全局的函数
    window[callbackName]=function(data){

    fn(data)

    }
    //动态的生成script
    //
    var scriptElem =document.createElement(“script”);
    scriptElem.src =url;
    document.body.appendChild(scriptElem);
    }
    crossDomain(“https://api.douban.com/v2/movie/in_theaters“,

    console.log(data);
    })

http的协议：

1、输入http://baidu.com,首先浏览器会找DNS解析服务器，找到对应的ip地址
2、浏览器和百度的服务器建立三次握手(为了确保数据可以可靠的交互)
3、把url地址加上请求的首部字段（用键值对的形式），比如请求能接受的文件类型(  content-type)、是否支持gzip压缩、用户代理<user-agent> </user-agent> 、接受的语言、如果是POST还包含了请求体包装成一个完整的请求报文发往百度服务器
4、百度服务器接受到请求报文之后进行解析，根据请求报文首部中的请求方法、请求地 址  标识去找到对应的文件资料，加上日期、响应的文件类型、编码、过期时间等包装  一个响 应的报文发往客户端
5、浏览器接受到响应报文之后，把内容渲染出来，如果是html文件，并且标签里面涉及 到  了link,src属性，会重新产生一个新的请求链接 (css解析器 html解析器 v8引擎)

Http的连接过程与性能优化

网络性能优化最主要的就是要减少HTTP请求及每次响应中内容的长度。可以从连接过程中的下列方面加以考虑：

1、域名解析
a)尽可能的减少域名解析次数 -- 少用外部链接文件(css/js)
2、创建连接
a)努力减少连接创建次数 -- 使用keep-alive避免重复连接
3、发送请求
a)尽力减少请求次数 - 合理设置expires时间、资源合并
4、等待响应
提高服务器端运行速度 - 提高数据运算及查询速度
5、接收响应
a)尽可能的减少响应数据长度 - 启用gzip或deflate压缩
